# 记忆迷宫小游戏

一个基于 HTML、CSS 和 JavaScript 实现的轻量级记忆闯关小游戏。玩家需要在短暂记忆迷宫路线后，凭记忆从起点走到终点。游戏围绕“规则简单、视觉干净、单局短、失败不挫败、有持续积累感”的体验目标设计，适合碎片时间放松与轻度挑战。

## 项目背景

《记忆迷宫》的目标用户画像来自功能设计文档：

- 用户：林子涵，28 岁，产品经理，工作 4 年
- 使用场景：高强度会议、文档和需求沟通后，希望用 1-2 分钟完成一局不费力但有反馈的小游戏
- 体验偏好：
  - 视觉干净，不需要复杂装饰
  - 规则一眼能懂：记住路线，走到终点
  - 有积累感：过关得奖励、看金额增长、接近提现目标
  - 失败不挫败：可复活、可重置、没有强惩罚表达
  - 随时可停：不依赖联网组队，也不需要长时间投入

游戏并不强调和别人竞争，而是用微小胜利感覆盖一天里的失控感。

## 核心玩法

1. 进入游戏后点击“开始游戏”。
2. 系统短暂展示迷宫路线。
3. 路线隐藏后，玩家使用方向键移动角色。
4. 在倒计时结束前走到终点即可过关。
5. 撞墙、超时或走错会失败，可选择复活或重置。
6. 过关后获得虚拟金额奖励，并进入更高难度关卡。

## 操作方式

| 操作 | 按键/入口 |
|---|---|
| 移动角色 | 方向键 ↑ ↓ ← → |
| 使用透视镜 | 数字键 1 |
| 使用时光暂停 | 数字键 2 |
| 使用复活甲 | 数字键 3 |
| 使用全息地图 | 数字键 4 |
| 重置关卡 | 页面内“重置”按钮 |
| 切换主题 | 右上角主题按钮 |

## 功能模块

### 1. 迷宫闯关

- 共 10 个难度递增关卡
- 关卡逐步增加迷宫尺寸，缩短记忆时间和通关时间
- 每关包含记忆阶段、操作阶段、成功/失败反馈
- 玩家需要在有限时间内从起点移动到终点

### 2. 奖励与提现目标

- 每关通关后获得虚拟金额奖励
- 页面展示累计金额、进度条和“满 50 元可提现”目标
- 提现功能为演示逻辑，实际支付能力需要后续接入支付接口

### 3. 道具系统

游戏内置道具商店，玩家可用虚拟金额购买并在关卡中使用：

| 道具 | 功能 |
|---|---|
| 透视镜 | 显示正确路径 3 秒 |
| 时光暂停 | 暂停倒计时 5 秒 |
| 复活甲 | 失败时自动复活 |
| 全息地图 | 显示迷宫缩略图 5 秒 |

### 4. 成就系统

成就用于增强长期反馈和完成感，包括：

- 首次过关
- 连续多关无失败
- 累计获得指定金额
- 剩余时间较多时通关
- 单关无撞墙通关
- 通关全部关卡
- 累计使用复活
- 累计购买道具

解锁成就后会获得额外虚拟金额奖励。

### 5. 每日签到

- 支持连续签到
- 每日签到获得虚拟金额奖励
- 第 7 天签到可获得额外复活甲奖励
- 签到数据保存在本地浏览器中

### 6. 排行榜

排行榜包含多个维度：

- 最快通关
- 最高金额
- 最少失败
- 最高成就

当前版本使用本地数据与模拟数据展示排行榜效果。

### 7. 分享功能

- 支持生成分享文案
- 支持复制分享内容
- 部分浏览器环境下可调用系统分享能力

### 8. 主题切换

- 支持深色/浅色主题
- 主题偏好保存在本地浏览器中
- 页面风格保持简洁、克制，强调低负担体验

## 核心数据流

```text
用户行为 → 游戏状态更新 → UI 重绘 → 本地存储
```

主要用户行为包括：

- 移动、撞墙、过关、失败
- 购买道具、使用道具
- 签到、分享、切换主题
- 查看成就、排行榜和奖励进度

本地存储内容包括：

- 主题偏好
- 道具库存
- 成就进度
- 签到状态
- 排行榜数据
- 玩家昵称

## 技术实现

- 使用单文件 `memory-maze.html` 承载主要结构、样式和脚本
- 使用 Canvas 绘制迷宫、路线、倒计时环和粒子背景
- 使用本地 GIF 资源展示玩家和终点
- 使用 `localStorage` 保存本地进度数据
- 使用 Font Awesome CDN 提供图标
- 无需后端服务，直接用浏览器打开即可运行

## 运行方式

直接用浏览器打开：

```text
memory-maze.html
```

也可以在项目目录启动一个本地静态服务：

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000/memory-maze.html
```

## 设计特点

- **规则低门槛**：记住路，走过去，一句话能理解
- **单局短反馈**：每局约 1-2 分钟，适合碎片时间
- **失败低压力**：提供复活、重置和道具缓冲
- **积累感明确**：金额、成就、签到、排行榜共同形成持续反馈
- **视觉克制**：深色/浅色主题切换，整体界面保持清爽
- **无需后端**：适合课程展示、原型演示和静态部署
